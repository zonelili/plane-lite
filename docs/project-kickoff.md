## 🎯 MVP 开发计划

### 功能范围

**核心流程**：注册 → 登录 → 创建项目 → 管理问题 → 看板查看

**5 大模块**：
1. 用户认证（注册、登录、JWT）
2. 工作区管理（简化版）
3. 项目管理（CRUD）
4. 问题管理（CRUD + 状态流转 + 评论）
5. 看板视图（按状态分组展示）

### 开发时间表

| 阶段 | 功能 | 预计工时 | 目标日期 |
|------|------|---------|----------|
| **Day 1** | 用户认证 + 工作区 | 4-6h | 2026-03-24 |
| **Day 2** | 项目管理 | 3-4h | 2026-03-25 |
| **Day 3-4** | 问题管理 | 8-10h | 2026-03-27 |
| **Day 5** | 看板视图 | 3-4h | 2026-03-28 |
| **Day 6-7** | 优化和部署 | 6-8h | 2026-03-30 |

**总工时**：24-32 小时（约 1 周）

## 📝 开发过程中的注意事项

### 1. 严格遵守约束

在 `CLAUDE.md` 中定义的 5 条非协商规则：
1. 分层架构（Controller → Service → Mapper）
2. 数据库规范（表名、字段、索引）
3. API 设计（RESTful、统一响应格式）
4. 前端规范（命名、路由）
5. 代码质量（事务、校验、测试）

### 2. 文档驱动开发

- 功能实现前先看文档
- 不清楚的地方问 AI
- 发现文档错误及时更新

### 3. 测试先行

- 编写测试用例
- 实现功能代码
- 验证测试通过

### 4. 小步快跑

- 每个功能点独立完成
- 完成后立即测试
- 确认无误再继续

### 5. 记录问题

- AI 理解偏差
- 代码质量问题
- 性能问题
- 用户体验问题
