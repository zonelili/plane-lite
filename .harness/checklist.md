## 📋 Pre-Flight Checklist

在开始 AI 开发之前，确保以下基础设施就绪：

### 1. 知识编码层（Knowledge Layer）

- [x] **CLAUDE.md 已创建**

  - 关键约束列出

- [x] **详细文档结构**
  - `/docs/requirements/` - 需求文档
  - `/docs/architecture/` - 架构设计
  - `/docs/conventions/` - 编码规范
  - `/docs/workflows/` - 工作流程
  - `/docs/decisions/` - 架构决策记录（ADR）

- [ ] **文档完整性**（待 AI 开发时补充）
  - [ ] 系统架构设计
  - [ ] 前端架构设计
  - [ ] 编码规范
  - [ ] 数据库规范
  - [ ] 前端规范

## 📝 开发流程检查

### 功能开发流程

- [ ] **1. 需求理解**
  - [ ] 阅读需求文档
  - [ ] 明确验收标准
  - [ ] 识别潜在风险

- [ ] **2. 技术设计**
  - [ ] 数据库设计（如需要）
  - [ ] API 接口设计
  - [ ] 前端组件设计
  - [ ] 创建 ADR（如涉及架构决策）

- [ ] **3. 测试驱动开发**
  - [ ] 编写失败的测试用例
  - [ ] 实现最小可用代码
  - [ ] 重构和优化
  - [ ] 确保测试通过

- [ ] **4. 代码审查**
  - [ ] 自我审查（代码规范、逻辑正确性）
  - [ ] 工具审查（Lint、测试覆盖率）
  - [ ] 同行审查（可选）

- [ ] **5. 集成和验证**
  - [ ] 本地运行验证
  - [ ] API 测试验证
  - [ ] UI 交互验证
  - [ ] 性能检查（如需要）

### Bug 修复流程

- [ ] **1. 问题重现**
  - [ ] 记录复现步骤
  - [ ] 确认影响范围
  - [ ] 定位问题根因

- [ ] **2. 测试先行**
  - [ ] 编写重现问题的测试用例
  - [ ] 确认测试失败

- [ ] **3. 修复问题**
  - [ ] 实现修复代码
  - [ ] 确认测试通过
  - [ ] 验证无副作用

## 🎯 质量标准

### 代码质量

- [ ] **后端代码**
  - [ ] 单元测试覆盖率 > 80%
  - [ ] Checkstyle 零告警
  - [ ] SonarQube 扫描通过

- [ ] **前端代码**
  - [ ] 单元测试覆盖率 > 60%
  - [ ] ESLint 零告警

- [ ] **交互流畅**
  - [ ] 操作响应及时
  - [ ] 加载状态明确
  - [ ] 错误提示友好
