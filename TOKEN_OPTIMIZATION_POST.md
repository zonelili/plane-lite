# 💡 如何优雅地减少 Claude Token 消耗？

> 不想浪费 token？这篇文章告诉你 3 个切实可行的方法。

---

## 问题是什么？

每次问 AI 一个看似简单的问题，结果 token 消耗翻倍？

这很可能是因为你的提问方式不够聪明。

**核心概念**：Token 消耗的本质就是"你让 AI 干了多少活"。要省 token，就是让 AI 干更聪明的活。

---

## 3 大方法省 Token

### 1️⃣ 模型切换 ⚡

选择合适的模型，而不是总用最强的。

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| **Haiku** | 便宜、快速 | 简单查询、代码片段、快速原型 |
| **Sonnet** | 平衡 | 一般开发任务、代码审查 |
| **Opus** | 强大、贵 | 复杂设计、多轮推理、关键决策 |

**核心逻辑**：
- Haiku 便宜 50%+，但可能需要多轮对话
- Opus 虽然贵，但通常一次搞定，最后反而可能省 token
- 建议：99% 情况用 Haiku，碰到"一次没解决要改第二次"的情况，考虑切到 Sonnet

---

### 2️⃣ 信息管理 📝

Token 消耗的核心就是"信息量"。让 AI 处理最少的信息，同时回答你最想要的问题。

#### A. 提问优化 - 让 AI 少读

**原则 1：具体描述问题**

```
❌ "帮我看看这个项目的代码质量怎么样"
✅ "统计一下这个项目中没有类型注解的函数有多少个"

❌ "这个代码为什么报错？"
✅ "运行这段代码报了 TypeError: undefined is not a function，
   这是什么原因？这是第 42 行的问题"
```

**原则 2：用工具替代阅读**

```
❌ 懒方式："帮我看看这个项目有没有提到 TypeScript"
✅ 聪明方式："搜一下这个项目中有没有 'typescript' 这个词"
   （AI 会用 grep，而不是读整个文档）

❌ 懒方式："这个文件夹有多少个 JavaScript 文件？"
✅ 聪明方式："统计一下有多少个 .js 文件"
   （AI 会用 find | wc -l，而不是列出所有文件）
```

**原则 3：分段读取大文件**

```
❌ 懒方式："帮我看看这个 10000 行的日志"
✅ 聪明方式："读一下这个文件第 100 到 150 行"
   （只读 50 行，而不是 10000 行）
```

**原则 4：一次问多个问题**

```
❌ 问题 1："搜一下项目中有没有 'deprecated'"
❌ 问题 2："统计一下代码行数"
❌ 问题 3："检查一下是否有 TODO 注释"

✅ 一次性问：
"帮我：
1. 搜一下项目中有没有 'deprecated'
2. 统计一下代码行数
3. 检查一下是否有 TODO 注释"
```

**省 token 的效果**：
- 具体提问 vs 模糊提问：省 60-80%
- 用工具搜索 vs 让 AI 读文件：省 90%+
- 分段读大文件 vs 读整个文件：省 80%
- 一次问多个 vs 分多次问：省 30%

#### B. 上下文管理 - 让 AI 少记

不是每次都让 AI 加载所有历史对话。

**`/compact` - 压缩长对话**

当对话变得很长时（50+ 轮对话），使用 `/compact` 命令压缩上下文。

```
什么时候用：
  • 对话超过 50 轮
  • 感觉每条消息都变慢了
  • 想继续聊同一话题，但对话历史太长

怎么用：
  直接输入 /compact

效果：
  ✓ 系统自动总结早期对话内容
  ✓ 保留重要信息和上下文
  ✓ 减少后续 token 消耗 40-60%
  ✓ 对话继续正常进行
```

**对话长度 vs 操作建议**：
```
< 50 轮    → 不用管（系统自动处理）
50-100 轮  → 考虑 /compact
> 100 轮   → 必须 /compact（否则每条消息都很贵）
```

**`/clear` - 清除所有对话**

当要完全切换话题时，使用 `/clear` 命令清除对话记录。

```
什么时候用：
  • 从项目 A 切换到完全不同的项目 B
  • 不再需要之前的上下文信息
  • 想从零开始一个新话题

怎么用：
  直接输入 /clear

效果：
  ✓ 彻底清除所有对话记录
  ✓ Token 计数完全重置
  ✓ 从零开始，不加载历史

注意：
  ⚠️ 清除后无法恢复
  ⚠️ 如果还可能用到之前的信息，先用 /compact 再 /clear
```

## 快速参考

**2 大方法的优先级**：

```
最有效果（能省 70%+）
  └─ 信息管理 - 提问优化 + 上下文压缩

持续效果（能省 20-30%）
  └─ 模型切换 - 根据任务复杂度选模型
```

**下次提问前的检查清单**：

- [ ] 我能用搜索/统计命令替代阅读吗？
- [ ] 我的问题足够具体吗？
- [ ] 我能一次性问多个相关问题吗？
- [ ] 对话是否已经很长了？→ 需要 /compact？
- [ ] 要完全切换话题吗？→ 需要 /clear？

---

**最重要的一点**：让 AI 做聪明活，不要让 AI 做苦力活。
