================================================================================
                            关键风险和应对
================================================================================

风险 1: 前后端字段名不一致 (snake_case vs camelCase)
  应对: 在 API 调用层进行字段转换

风险 2: 网络错误和超时
  应对: 实现重试机制，显示友好提示

风险 3: 加载状态管理不当导致重复提交
  应对: 提交前检查 loading 状态，禁用按钮

风险 4: Token 过期处理
  应对: Axios 拦截器自动捕获 401，清除 Token 并重定向登录

风险 5: 数据同步问题
  应对: 编辑/删除后重新加载，冲突时显示提示

================================================================================
                          重要决策和设计
================================================================================

1. 字段名转换
   前端使用 camelCase，后端使用 snake_case
   在 API 层统一处理转换

2. 分页策略
   使用传统分页 (page + size)，不用无限滚动
   前端维护分页状态，后端返回 items + pagination

3. 错误处理
   401 → 重定向登录
   403/404 → 显示友好提示
   500 → 显示服务器错误
   Network → 显示网络错误

4. 加载状态
   显示加载动画或骨架屏
   禁用操作按钮防止重复提交
   保存用户输入避免丢失

5. 表单验证
   客户端：字段长度、格式校验
   提交前：完整表单验证
   服务器：后端再验证一遍
